## 1. 문서 개요

### 1.1 제품 정의
- 컷메이트 AI는 영상 원본을 분석해 자막, 컷 편집 후보, 하이라이트 구간, 썸네일 후보를 제안하는 AI 기반 영상 편집 보조 시스템이다.
- 사용자는 AI가 만든 편집 초안을 검토하고 승인, 수정, 거절할 수 있다.
- MVP는 완전 자동 편집 도구가 아니라 편집자의 반복 작업을 줄이는 협업형 보조 도구를 목표로 한다.
- MVP는 외부 유료 API 호출을 기본 전제로 두지 않고, 로컬에서 실행 가능한 오픈소스 모델과 영상 처리 도구를 우선 사용한다.

### 1.2 MVP 목표
- 20분 이하 영상 기준으로 첫 편집 초안과 썸네일 후보를 생성한다.
- 평균 사용 영상은 5~10분으로 예상하고, 이 구간에서 분석 대기 피로를 최소화한다.
- 사용자가 자막, 컷 후보, 하이라이트 후보, 썸네일 후보를 한 흐름에서 검토할 수 있게 한다.
- 최종 영상과 선택한 썸네일 이미지를 다운로드할 수 있게 한다.

### 1.3 MVP 범위
- 포함: 영상 업로드, 기본 메타데이터 추출, 음성 인식, 자막 초안, 컷 후보 탐지, 하이라이트 추천, 썸네일 후보 추천, 타임라인 검토, 결과물 내보내기
- 포함: 로컬 음성 인식, 로컬 장면 분석, 로컬 썸네일 후보 점수화, 로컬 추천 근거 생성
- 제외: 완전 자동 게시, 실시간 라이브 편집, 전문 색보정, 전문 사운드 믹싱, AI 생성형 썸네일 이미지, 딥페이크/음성 복제/얼굴 합성, 필수 외부 API 의존 구조

## 2. 사용자 및 권한

### 2.1 주요 사용자
- 1인 크리에이터: 브이로그, 리뷰, 숏폼 클립을 빠르게 제작하려는 사용자
- 교육 콘텐츠 제작자: 긴 강의 영상의 자막과 핵심 구간을 정리하려는 사용자
- 마케팅 콘텐츠 담당자: 업로드 가능한 영상과 썸네일 후보를 함께 확보하려는 사용자

### 2.2 권한 요구사항
| ID | 요구사항 | 우선순위 |
|---|---|---|
| AUTH-001 | 사용자는 본인이 업로드한 프로젝트만 조회, 수정, 다운로드할 수 있어야 한다. | Must |
| AUTH-002 | 공유/협업 기능은 MVP 범위에서 제외한다. | Won't |
| AUTH-003 | 사용자는 프로젝트 삭제를 요청할 수 있어야 하며, 삭제된 프로젝트의 원본/산출물 접근은 차단되어야 한다. | Must |

## 3. 기능 요구사항

### 3.1 영상 업로드
| ID | 요구사항 | 상세 설명 | 우선순위 | 검수 기준 |
|---|---|---|---|---|
| UP-001 | 영상 파일 업로드 | 사용자는 영상 파일을 업로드할 수 있다. | Must | 업로드 완료 후 프로젝트가 생성된다. |
| UP-002 | 파일 형식 제한 | MVP는 `mp4`, `mov`, `m4v` 파일을 우선 지원한다. | Must | 미지원 파일 업로드 시 사유를 안내한다. |
| UP-003 | 용량/길이 제한 | MVP는 20분 이하 영상과 사전에 정의된 최대 용량을 기준으로 처리한다. | Must | 20분 초과 또는 용량 제한 초과 시 분석 시작 전에 이유를 안내한다. |
| UP-004 | 기본 정보 추출 | 시스템은 길이, 해상도, 프레임 비율, 오디오 포함 여부를 추출한다. | Must | 프로젝트 상세에서 기본 정보가 표시된다. |
| UP-005 | 콘텐츠 목적 선택 | 사용자는 숏폼, 브이로그, 강의, 인터뷰, 홍보 영상 중 목적을 선택할 수 있다. | Must | 선택값이 분석 요청에 포함된다. |

| JOB-004 | 비동기 처리 | 영상 분석과 렌더링은 비동기 작업으로 처리한다. | Must | 사용자는 작업 상태를 새로고침 없이 확인할 수 있다. |

### 3.9 로컬 처리 및 모델 관리
| ID | 요구사항 | 상세 설명 | 우선순위 | 검수 기준 |
|---|---|---|---|---|
| LOC-001 | 로컬 우선 처리 | 기본 MVP 분석 파이프라인은 외부 유료 API 없이 로컬에서 실행 가능해야 한다. | Must | 네트워크 API 키 없이 샘플 영상 분석이 가능하다. |
| LOC-002 | 영상 처리 도구 | 시스템은 FFmpeg 기반으로 오디오, 프레임, 저해상도 프록시를 추출한다. | Must | 업로드 영상에서 분석용 오디오와 프레임 샘플이 생성된다. |
| LOC-003 | 로컬 음성 인식 | 시스템은 faster-whisper 기반 로컬 음성 인식을 지원한다. | Must | 오디오 파일에서 타임코드 포함 자막 초안이 생성된다. |

| LOC-005 | 장면 분석 | 시스템은 PySceneDetect 또는 OpenCV 기반 장면 전환 탐지를 지원한다. | Must | 장면 전환 지점과 후보 프레임이 생성된다. |
| LOC-006 | 썸네일 품질 점수화 | 시스템은 OpenCV로 블러, 밝기, 구도, 크롭 안정성을 점수화한다. | Must | 부적합 프레임이 기본 썸네일 후보에서 제외된다. |
| LOC-007 | 썸네일 의미 매칭 | 시스템은 OpenCLIP 기반으로 키워드/하이라이트와 후보 프레임의 의미 유사도를 계산할 수 있다. | Should | 하이라이트 키워드와 관련 높은 프레임이 상위 후보에 배치된다. |
| LOC-008 | 로컬 추천 근거 생성 | 시스템은 Qwen3-8B, gpt-oss-20b 등 로컬 LLM을 사용해 추천 이유를 생성할 수 있다. | Should | 추천 구간에 자연어 추천 근거가 생성된다. |
| LOC-009 | 로컬 VLM 확장 | 시스템은 Qwen3-VL-8B 등 로컬 VLM으로 썸네일 후보 설명을 보강할 수 있다. | Could | VLM이 꺼져도 기본 후보 추천은 동작한다. |
| LOC-010 | API 선택 옵션 | 외부 API는 필수 경로가 아니라 고품질 분석 옵션으로만 제공한다. | Could | API 설정이 없어도 MVP 핵심 기능이 동작한다. |

## 4. 화면 요구사항

### 4.1 업로드 화면
- 파일 업로드 영역을 제공한다.
- 지원 파일 형식, 최대 길이, 최대 용량을 업로드 전에 안내한다.
- 콘텐츠 목적과 출력 목표를 선택할 수 있어야 한다.
- 제한 초과 또는 미지원 파일은 분석 전에 차단한다.

### 4.2 분석 진행 화면
- 업로드 완료, 음성 인식, 장면 분석, 추천 생성, 렌더링 단계를 표시한다.
- 예상 완료 시간을 표시하되, 정확하지 않을 경우 "예상"임을 명확히 한다.
- 실패 시 실패 단계, 원인, 재시도 액션을 제공한다.

### 4.3 편집 초안 화면
- 영상 미리보기, AI 추천 타임라인, 자막 패널을 함께 제공한다.
- 컷/하이라이트/자막 추천은 구분 가능한 시각적 라벨을 가진다.
- 추천 근거는 구간 선택 시 즉시 확인할 수 있어야 한다.
- 사용자는 추천을 승인, 수정, 삭제, 원복할 수 있어야 한다.

### 4.4 썸네일 추천 화면
- 썸네일 후보는 카드 형태로 3~5개 표시한다.
- 각 카드에는 이미지, 타임스탬프, 추천 이유, 선택 버튼을 제공한다.
- 직접 프레임 선택을 위한 영상 스크러버를 제공한다.
- 플랫폼별 크롭 미리보기는 후보 선택 전에 확인 가능해야 한다.

### 4.5 내보내기 화면
- 비율, 해상도, 자막 포함 여부, 썸네일 포함 여부를 선택할 수 있어야 한다.
- 렌더링 상태와 다운로드 버튼을 제공한다.
- 실패한 렌더링 작업은 재시도할 수 있어야 한다.

## 6. 로컬 AI 분석 요구사항

### 6.0 로컬 처리 원칙
- MVP의 기본 분석은 로컬 실행을 기준으로 한다.
- 비용 절감과 개인정보 보호를 위해 원본 영상, 오디오, 프레임을 외부 API로 전송하지 않는 구성을 기본값으로 둔다.
- 로컬 모델은 작업별로 분리한다. 음성 인식, 장면 분석, 썸네일 점수화, 추천 이유 생성은 서로 다른 도구/모델이 담당한다.
- 사용자의 장비 성능에 따라 경량 모드와 고품질 모드를 선택할 수 있도록 확장 가능해야 한다.

### 6.0.1 기본 모델/도구 구성

| 영역 | 기본 도구/모델 | 우선순위 |
|---|---|---|
| 영상/오디오 추출 | FFmpeg | Must |
| 음성 인식 | faster-whisper | Must |
| 무음/발화 탐지 | Silero VAD 또는 오디오 레벨 분석 | Must |
| 장면 전환 | PySceneDetect, OpenCV | Must |
| 썸네일 품질 분석 | OpenCV | Must |
| 썸네일 의미 매칭 | OpenCLIP | Should |
| 추천 이유 생성 | Qwen3-8B, gpt-oss-20b 등 로컬 LLM | Should |
| 이미지 설명 보강 | Qwen3-VL-8B 등 로컬 VLM | Could |
| 화자 분리 | pyannote.audio 또는 WhisperX | Could |

### 6.1 입력
- 영상 파일
- 오디오 트랙
- 프레임 샘플
- 콘텐츠 목적
- 출력 목표
- 제목/주요 키워드(선택)
- 브랜드 톤 또는 금지 표현(선택)

### 6.2 출력
- 타임코드 기반 자막 초안
- 삭제 추천 구간 목록
- 하이라이트 추천 구간 목록
- 썸네일 후보 목록
- 추천 근거와 내부 점수
- 로컬 처리 로그와 사용한 모델/도구 버전

### 6.3 품질 기준
- 추천 결과에는 사용자가 납득할 수 있는 근거가 있어야 한다.
- 컷 후보와 하이라이트 후보는 서로 충돌하지 않아야 한다.
- 썸네일 후보는 영상 실제 프레임에서만 생성되어야 한다.
- 민감 정보, 부적절한 장면, 왜곡 소지가 있는 프레임은 썸네일 후보에서 제외한다.
- 로컬 LLM 또는 VLM이 비활성화되어도 규칙 기반 후보 탐지와 기본 점수화는 동작해야 한다.
