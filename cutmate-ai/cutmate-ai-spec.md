# 컷메이트 AI 제품 스펙

> **문서 버전:** v0.1 (Spec Draft)  
> **작성일:** 2026. 05. 29  
> **기준 문서:** `cutmate-ai-planning.md`, `cutmate-ai-requirements.md`  
> **목적:** 컷메이트 AI MVP의 제품 범위, 사용자 흐름, 기능 동작, 데이터, 예외 처리, 수용 기준을 하나의 실행 가능한 스펙으로 정리한다.

---

## 1. 제품 요약

컷메이트 AI는 영상 원본을 업로드하면 AI가 자동 자막, 삭제 추천 구간, 하이라이트 구간, 썸네일 후보를 생성하고, 사용자가 이를 검토해 최종 영상과 썸네일 이미지를 다운로드할 수 있게 하는 영상 편집 보조 시스템이다.

MVP는 완전 자동 편집기가 아니라 편집자의 반복 작업을 줄이는 협업형 도구다. 모든 AI 추천은 자동 적용하지 않고, 사용자의 승인/수정/거절 과정을 거친다.

제품의 기본 방향은 로컬 우선이다. 외부 유료 API를 필수 전제로 두지 않고, 사용자의 로컬 머신 또는 개인 서버에서 FFmpeg, faster-whisper, OpenCV, PySceneDetect, OpenCLIP, 로컬 LLM을 조합해 무료로 실행 가능한 분석 파이프라인을 제공한다.

## 2. 핵심 결정사항

| 항목 | 결정 |
|---|---|
| 프로젝트명 | 컷메이트 AI |
| 주요 사용자 | 1인 크리에이터, 교육 콘텐츠 제작자, 마케팅 콘텐츠 담당자 |
| 평균 영상 길이 | 5~10분 |
| 최대 영상 길이 | 20분 이하 |
| 20분 초과 영상 | 업로드/분석 차단, 처리 시간과 비용 증가 이유 안내 |
| 지원 파일 형식 | `mp4`, `mov`, `m4v` 우선 |
| 썸네일 방식 | 영상 속 실제 프레임 기반 후보 추천 |
| 생성형 썸네일 | MVP 제외 |
| 썸네일 점수 노출 | 사용자에게 숫자 점수는 숨기고 추천 이유만 노출 |

| 프로젝트 삭제 | 사용자에게 즉시 삭제 상태로 보이고, 원본/산출물은 24시간 이내 물리 삭제 |
| 기본 AI 전략 | 로컬 오픈소스 모델/도구 우선 |
| API 사용 | 필수 경로가 아니라 향후 고품질 분석 옵션 |
| 기본 음성 인식 | faster-whisper |
| 기본 영상 분석 | FFmpeg, PySceneDetect, OpenCV |
| 기본 썸네일 분석 | OpenCV 품질 점수 + OpenCLIP 의미 매칭 |
| 추천 이유 생성 | Qwen3-8B, gpt-oss-20b 등 로컬 LLM 선택 |

## 3. MVP 목표

- 업로드한 20분 이하 영상에서 첫 편집 초안과 썸네일 후보를 생성한다.
- 사용자가 자막, 컷 후보, 하이라이트 후보, 썸네일 후보를 한 작업 흐름에서 검토할 수 있게 한다.
- 사용자는 AI 추천을 승인, 수정, 거절하거나 건너뛸 수 있다.
- 최종 영상과 선택한 썸네일 이미지를 다운로드할 수 있다.
- 20분 초과 영상은 분석을 시작하지 않고, 차단 이유와 사용자가 취할 수 있는 대안을 안내한다.
- API 키 없이도 핵심 분석 기능이 동작하는 로컬 MVP를 제공한다.

## 4. MVP 범위

### 4.1 포함 범위
- 영상 업로드 및 기본 메타데이터 추출
- 분석 진행 상태 표시
- 음성 인식 기반 자동 자막 생성
- 자막 문구 수정 및 싱크 조정
- 무음/반복/불필요 구간 탐지
- 하이라이트 구간 추천
- AI 추천 타임라인 표시
- 썸네일 후보 3~5개 추천
- 직접 썸네일 프레임 선택
- 원본 비율 및 9:16 숏폼 결과물 내보내기
- 선택한 썸네일 이미지 다운로드
- 로컬 모델/도구 기반 분석 파이프라인
- 경량/기본/고품질 분석 모드 설계

### 4.2 제외 범위
- 완전 자동 게시
- 실시간 라이브 편집
- 전문 색보정 및 전문 사운드 믹싱
- AI 생성형 썸네일 이미지
- 딥페이크, 음성 복제, 얼굴 합성
- 저작권이 불명확한 외부 음원 자동 삽입
- 팀 협업 리뷰/코멘트
- BGM 자동 추천
- 브랜드 템플릿 기반 썸네일 편집
- 외부 API를 필수로 요구하는 분석 기능

## 5. 사용자 흐름

### 5.1 기본 성공 흐름
1. 사용자가 영상 파일을 업로드한다.
2. 시스템은 파일 형식, 영상 길이, 용량, 오디오 포함 여부를 검사한다.
3. 영상이 20분 이하이고 지원 형식이면 프로젝트를 생성한다.
4. 사용자가 콘텐츠 목적과 출력 목표를 선택한다.
5. 시스템은 음성 인식, 장면 분석, 컷 후보, 하이라이트 후보, 썸네일 후보 생성을 진행한다.
6. 사용자는 편집 초안 화면에서 자막, 컷 후보, 하이라이트 후보를 검토한다.
7. 사용자는 썸네일 추천 화면에서 후보를 선택하거나 직접 프레임을 지정한다.
8. 사용자는 내보내기 설정에서 비율, 해상도, 자막 포함 여부, 썸네일 포함 여부를 선택한다.
9. 시스템은 결과물을 렌더링하고 다운로드 URL을 제공한다.

### 5.2 20분 초과 영상 흐름
1. 사용자가 20분을 초과하는 영상을 업로드한다.
2. 시스템은 분석 작업을 생성하지 않는다.
3. 사용자는 다음 안내를 확인한다.

```text
현재는 20분 이하 영상만 분석할 수 있습니다.
컷메이트 AI MVP는 평균 5~10분 영상에 최적화되어 있어, 긴 영상은 처리 시간과 비용이 크게 증가할 수 있습니다.
영상을 20분 이하로 분할하거나 핵심 구간만 잘라 다시 업로드해 주세요.
```

4. 사용자는 파일을 제거하고 다른 영상을 업로드할 수 있다.

### 5.3 오디오 없는 영상 흐름
1. 사용자가 오디오가 없는 영상을 업로드한다.
2. 시스템은 자막 생성 단계를 건너뛰고 장면/프레임 분석을 진행한다.
3. 사용자는 자막 없이 컷 후보, 하이라이트 후보, 썸네일 후보를 확인한다.

## 6. 화면 스펙

### 6.1 업로드 화면
- 파일 업로드 영역을 제공한다.
- 지원 형식은 `mp4`, `mov`, `m4v`로 안내한다.
- 평균 권장 길이는 5~10분, 최대 길이는 20분으로 안내한다.
- 20분 초과 영상은 분석 전에 차단한다.
- 콘텐츠 목적 선택값을 제공한다: 숏폼, 브이로그, 강의, 인터뷰, 홍보 영상.
- 출력 목표 선택값을 제공한다: 원본 요약, 하이라이트 추출, 자막 생성, 숏폼 변환.

### 6.2 분석 진행 화면
- 업로드 완료, 음성 인식, 장면 분석, 추천 생성, 렌더링 단계를 표시한다.
- 단계별 진행률과 현재 상태를 표시한다.
- 예상 완료 시간은 "예상" 값임을 명확히 표시한다.
- 실패 시 실패 단계, 원인, 재시도 버튼을 제공한다.

### 6.3 편집 초안 화면
- 영상 미리보기를 제공한다.
- 추천 컷, 하이라이트, 자막을 타임라인에 표시한다.
- 추천 구간은 대기, 승인, 거절, 수정 상태를 가진다.
- 사용자는 컷 후보를 승인하거나 거절할 수 있다.
- 사용자는 자막 문구와 싱크를 수정할 수 있다.
- 구간 선택 시 추천 근거를 확인할 수 있다.
- 사용자는 특정 구간을 썸네일 후보 검토 대상으로 보낼 수 있다.

### 6.4 썸네일 추천 화면
- 썸네일 후보 3~5개를 카드로 표시한다.
- 각 후보는 이미지, 타임스탬프, 추천 이유, 선택 버튼을 포함한다.
- 사용자에게 내부 점수 숫자는 노출하지 않는다.
- 추천 이유 예시는 다음과 같다.
  - 핵심 장면과 연결됨
  - 제품이 화면 중앙에 노출됨
  - 화자 표정이 자연스러움
  - 구도가 안정적이고 선명함
- 사용자는 직접 프레임 선택 타임라인 스크러버로 후보를 대체할 수 있다.
- 플랫폼별 미리보기는 16:9, 9:16, 1:1을 우선 제공한다.

### 6.5 내보내기 화면
- 비율을 선택할 수 있다: 원본 비율, 9:16, 1:1.
- 해상도를 선택할 수 있다.
- 자막 포함 여부를 선택할 수 있다.
- 썸네일 이미지 포함 여부를 선택할 수 있다.
- 렌더링 상태를 표시한다: 대기, 진행, 완료, 실패.
- 완료 시 영상 다운로드 URL과 썸네일 다운로드 URL을 제공한다.

## 7. 기능 스펙

### 7.1 영상 업로드
| ID | 스펙 | 수용 기준 |
|---|---|---|
| UP-001 | 사용자는 영상 파일을 업로드할 수 있다. | 지원 파일이면 프로젝트가 생성된다. |
| UP-002 | 시스템은 파일 형식을 검사한다. | `mp4`, `mov`, `m4v` 외 파일은 차단하고 이유를 보여준다. |
| UP-003 | 시스템은 영상 길이를 검사한다. | 20분 초과 영상은 분석 작업을 생성하지 않는다. |
| UP-004 | 시스템은 영상 기본 정보를 추출한다. | 길이, 해상도, 비율, 오디오 포함 여부가 저장된다. |

### 7.2 분석 작업
| ID | 스펙 | 수용 기준 |
|---|---|---|
| JOB-001 | 업로드 완료 후 분석 작업을 생성한다. | 작업 상태가 `queued`로 저장된다. |
| JOB-002 | 분석은 비동기로 처리한다. | 사용자는 진행 상태를 화면에서 확인할 수 있다. |
| JOB-003 | 실패 시 실패 단계와 원인을 저장한다. | 사용자는 재시도 가능 여부를 확인할 수 있다. |

### 7.3 자동 자막
| ID | 스펙 | 수용 기준 |
|---|---|---|
| SUB-001 | 오디오를 텍스트로 변환한다. | 타임코드가 있는 자막 세그먼트가 생성된다. |
| SUB-002 | 자막은 문장 또는 의미 단위로 분리한다. | 각 세그먼트에 시작/종료 시간이 있다. |
| SUB-003 | 사용자는 자막 문구를 수정할 수 있다. | 수정 내용이 저장되고 미리보기에 반영된다. |
| SUB-004 | 사용자는 자막 싱크를 조정할 수 있다. | 조정된 시간이 타임라인에 반영된다. |

### 7.4 컷 후보 탐지
| ID | 스펙 | 수용 기준 |
|---|---|---|
| CUT-001 | 긴 무음 구간을 삭제 후보로 표시한다. | 후보에 시작/종료 시간과 이유가 포함된다. |
| CUT-002 | 반복 발화, 말 더듬음, 대기 구간을 후보로 표시한다. | 후보별 추천 근거가 표시된다. |
| CUT-003 | 컷 후보는 자동 삭제하지 않는다. | 사용자가 승인해야 편집안에 반영된다. |
| CUT-004 | 사용자는 승인한 컷을 원복할 수 있다. | 원복 후 타임라인과 미리보기가 갱신된다. |

### 7.5 하이라이트 추천
| ID | 스펙 | 수용 기준 |
|---|---|---|
| HIL-001 | 콘텐츠 목적에 맞는 핵심 구간을 추천한다. | 1개 이상의 후보 구간이 생성된다. |
| HIL-002 | 추천 이유를 함께 제공한다. | 사용자는 왜 추천됐는지 확인할 수 있다. |
| HIL-003 | 선택된 하이라이트는 숏폼 결과물 생성에 사용된다. | 선택 구간 기반 9:16 내보내기가 가능하다. |

### 7.6 썸네일 후보 추천
| ID | 스펙 | 수용 기준 |
|---|---|---|
| THM-001 | 영상 실제 프레임에서 후보 3~5개를 생성한다. | 후보 이미지와 타임스탬프가 표시된다. |
| THM-002 | 후보는 하이라이트 연관성, 선명도, 구도, 주제 노출, 안전성을 기준으로 선정한다. | 추천 이유가 후보별로 저장된다. |
| THM-003 | 흐림, 눈 감음, 과도한 크롭, 민감 정보 노출 프레임은 제외한다. | 부적합 후보가 기본 목록에 노출되지 않는다. |
| THM-004 | 사용자는 후보 중 하나를 선택할 수 있다. | 선택 후보가 프로젝트 산출물로 저장된다. |
| THM-005 | 사용자는 직접 프레임을 지정할 수 있다. | 직접 지정한 프레임이 썸네일 이미지로 저장된다. |
| THM-006 | 숫자 점수는 사용자에게 노출하지 않는다. | UI에는 추천 이유만 표시된다. |

### 7.7 결과물 내보내기
| ID | 스펙 | 수용 기준 |
|---|---|---|
| EXP-001 | 사용자는 원본 비율 영상을 다운로드할 수 있다. | 렌더링 완료 후 다운로드 URL이 제공된다. |
| EXP-002 | 사용자는 9:16 숏폼 영상을 다운로드할 수 있다. | 중앙 크롭 기본값과 사용자 조정값이 반영된다. |
| EXP-003 | 사용자는 자막 포함 여부를 선택할 수 있다. | 선택값에 따라 렌더링 결과가 달라진다. |
| EXP-004 | 사용자는 선택한 썸네일 이미지를 다운로드할 수 있다. | 썸네일 다운로드 URL이 제공된다. |

### 7.8 로컬 처리 및 모델 관리
| ID | 스펙 | 수용 기준 |
|---|---|---|
| LOC-001 | 기본 분석 파이프라인은 외부 API 키 없이 실행된다. | 샘플 영상 분석이 로컬 모델/도구만으로 완료된다. |
| LOC-002 | FFmpeg로 분석용 오디오, 프레임, 프록시 영상을 추출한다. | 분석 작업 시작 시 중간 산출물이 생성된다. |
| LOC-003 | faster-whisper로 로컬 자막을 생성한다. | 타임코드 포함 자막 초안이 생성된다. |
| LOC-004 | Silero VAD 또는 오디오 레벨 분석으로 발화/무음 구간을 탐지한다. | 무음 후보가 컷 후보로 연결된다. |
| LOC-005 | PySceneDetect/OpenCV로 장면 전환과 후보 프레임을 추출한다. | 장면 전환 지점과 썸네일 후보 프레임이 생성된다. |
| LOC-006 | OpenCV로 썸네일 후보의 품질을 점수화한다. | 흐림, 어두움, 불안정한 크롭 후보가 제외된다. |
| LOC-007 | OpenCLIP으로 하이라이트 키워드와 프레임 의미 유사도를 계산한다. | 의미 관련성이 높은 후보가 상위에 배치된다. |
| LOC-008 | 로컬 LLM으로 추천 이유를 생성한다. | LLM이 비활성화되어도 기본 추천 후보는 생성된다. |
| LOC-009 | 외부 API는 선택형 고품질 분석 옵션으로만 둔다. | API 설정이 없어도 Must 기능이 동작한다. |

## 8. 로컬 AI 처리 스펙

### 8.0 처리 원칙
- 대형 API 모델 하나에 모든 처리를 맡기지 않고, 작업별 전용 도구를 연결한다.
- 후보 탐지와 점수화는 최대한 결정적 규칙과 신호 기반 분석으로 처리한다.
- 로컬 LLM은 추천 후보를 만드는 핵심 의사결정자라기보다, 후보 이유를 자연어로 정리하고 자막 문장을 다듬는 보조 역할로 둔다.
- 로컬 LLM/VLM이 설치되지 않아도 자막, 컷 후보, 기본 썸네일 후보는 생성되어야 한다.
- 외부 API 연동은 기본값이 아니라 사용자가 명시적으로 켜는 선택 기능으로 둔다.

### 8.0.1 기본 로컬 모델/도구 구성

| 처리 영역 | 기본 구성 | 역할 | 적용 단계 |
|---|---|---|---|
| 영상/오디오 추출 | FFmpeg | 오디오, 프레임, 저해상도 프록시 생성 | MVP 필수 |
| 음성 인식 | faster-whisper | 자막 생성, 타임코드 추출 | MVP 필수 |
| 무음/발화 탐지 | Silero VAD 또는 오디오 레벨 분석 | 침묵/발화 구간 분리 | MVP 필수 |
| 장면 전환 | PySceneDetect, OpenCV | 컷 포인트, 장면 변화, 후보 프레임 추출 | MVP 필수 |
| 썸네일 품질 분석 | OpenCV | 블러, 밝기, 구도, 크롭 안정성 점수화 | MVP 필수 |
| 썸네일 의미 매칭 | OpenCLIP | 키워드/하이라이트와 프레임 의미 유사도 계산 | MVP 권장 |
| 추천 이유 생성 | Qwen3-8B, gpt-oss-20b 등 로컬 LLM | 하이라이트/썸네일 추천 이유 문장화 | 2차 |
| 이미지 설명 보강 | Qwen3-VL-8B 등 로컬 VLM | 썸네일 후보 장면 설명 보강 | 후순위 |
| 화자 분리 | pyannote.audio 또는 WhisperX | 인터뷰/강의 화자 구분 | 후순위 |

### 8.2 처리 단계
1. 영상 메타데이터 추출
2. 파일 형식, 길이, 용량 검증
3. FFmpeg로 오디오와 분석용 프레임/프록시 추출
4. faster-whisper로 음성 인식 및 타임코드 생성
5. 자막 세그먼트 생성
6. Silero VAD 또는 오디오 레벨 분석으로 무음/발화 구간 탐지
7. 반복 발화, 말 더듬음, 대기 구간 후보 탐지
8. PySceneDetect/OpenCV로 장면 전환과 후보 프레임 추출
9. 주요 키워드, 발화 밀도, 장면 변화 기준으로 하이라이트 후보 점수화
10. OpenCV로 썸네일 후보의 선명도, 밝기, 구도, 크롭 안정성 점수화
11. OpenCLIP으로 하이라이트 키워드와 후보 프레임의 의미 유사도 계산
12. 로컬 LLM으로 컷/하이라이트/썸네일 추천 근거 생성
13. 편집 타임라인 초안 생성

## 15. 검수 체크리스트

- 20분 이하 영상은 업로드 후 분석 작업이 생성된다.
- 20분 초과 영상은 분석 작업이 생성되지 않고 이유가 안내된다.
- 오디오가 있는 영상은 자막 초안이 생성된다.
- 오디오가 없는 영상은 자막 없이 장면 분석을 진행한다.
- API 키가 없어도 로컬 분석 파이프라인이 동작한다.
- 로컬 LLM이 없어도 기본 자막, 컷 후보, 썸네일 후보가 생성된다.
- 컷 후보는 자동 삭제되지 않는다.
- 하이라이트 후보에는 추천 근거가 표시된다.
- 썸네일 후보는 3~5개 표시된다.
- 썸네일 후보의 내부 점수는 UI에 표시되지 않는다.
- 사용자는 직접 썸네일 프레임을 지정할 수 있다.
- 사용자는 원본 비율 또는 9:16 결과물을 다운로드할 수 있다.
- 프로젝트 삭제 후 사용자는 원본/산출물에 접근할 수 없다.

## 16. 남은 결정사항

- MVP 최대 파일 용량을 몇 MB로 제한할지 결정해야 한다.
- 20분 초과 안내 화면에서 영상 분할 가이드를 얼마나 자세히 제공할지 결정해야 한다.
