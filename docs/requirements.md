# 요구사항 정의서 (PRD: Product Requirement Document)

> **문서 버전:** v1.0 (MVP)  
> **작성일:** 2026. 05. 26  
> **대상:** 프론트엔드, 백엔드, AI 엔지니어

---

## 1. 요구사항 ID 규칙
* **FE**: Frontend (화면 및 클라이언트 로직)
* **BE**: Backend (서버, DB, API)
* **AI**: AI Core (프롬프트, RAG, 모델 서빙)

---

## 2. 상세 요구사항 명세

### 2.1 기업/JD 기반 질문 생성 (Phase 1-1)
| 요구사항 ID | 기능명 | 상세 요구사항 (Description) | 우선순위 | 담당 |
| :--- | :--- | :--- | :--- | :--- |
| **REQ-JD-001** | 기업/JD 정보 입력 | 유저가 목표로 하는 기업명, 직무 카테고리, JD(텍스트)를 입력할 수 있는 폼을 제공한다. | Must | FE/BE |
| **REQ-AI-001** | 직무 키워드 및 질문 추출 | 입력된 JD를 분석하여 핵심 역량 키워드를 도출하고, 이에 매핑되는 면접 질문 1개를 생성한다. | Must | AI |
| **REQ-AI-002** | 출제 의도(Meta) 생성 | 질문 생성 시, 유저의 납득감을 위해 "왜 이 질문을 출제했는지"에 대한 근거 문장을 함께 생성하여 넘겨준다. | Must | AI |

### 2.2 텍스트 및 STT 보조 입력 (Phase 1-2)
| 요구사항 ID | 기능명 | 상세 요구사항 (Description) | 우선순위 | 담당 |
| :--- | :--- | :--- | :--- | :--- |

| **REQ-FE-002** | 브라우저 STT 보조 | 데스크톱 크롬 환경을 기준으로 `Web Speech API`를 활용한 단방향 음성 인식 받아쓰기 버튼을 제공한다. | Must | FE |
| **REQ-FE-003** | 제출 전 텍스트 검토 | STT로 변환되거나 타이핑된 줄글을 유저가 최종 제출하기 전에 자유롭게 수정/편집할 수 있어야 한다. | Must | FE |

### 2.3 꼬리 질문 생성 및 비용 통제 (Phase 1-3)
| 요구사항 ID | 기능명 | 상세 요구사항 (Description) | 우선순위 | 담당 |
| :--- | :--- | :--- | :--- | :--- |
| **REQ-AI-003** | 경량 모델 기반 꼬리질문 | 유저의 1차 답변의 STAR 논리 구조를 점검하고, 다음 답변 개선을 유도하는 후속 꼬리질문 1개를 생성한다. (비용 절감을 위해 경량 모델 활용) | Must | AI |
| **REQ-BE-001** | 세션 컨텍스트 요약 | 꼬리질문이 이어질 때 토큰 폭발을 막기 위해, 이전 대화 기록 전체를 넘기지 않고 이전 답변의 요약본(Summary)만 컨텍스트에 누적하여 AI에 전달한다. | Must | BE/AI |
| **REQ-BE-002** | 턴(Turn) 제한 가드레일 | MVP 단계에서 무한 꼬리질문을 막기 위해 [기본 질문 1개 + 꼬리 질문 최대 2개]로 세션 길이를 강제 제한한다. | Must | BE |

### 2.4 STAR 기반 답변 분석 및 리포트 (Phase 1-4)
| 요구사항 ID | 기능명 | 상세 요구사항 (Description) | 우선순위 | 담당 |
| :--- | :--- | :--- | :--- | :--- |
| **REQ-AI-004** | Structured JSON 출력 | AI는 유저의 최종 답변을 Few-Shot 루브릭에 따라 S/T/A/R 컴포넌트로 강제 쪼개고, 각 파트별 점수와 매칭 텍스트를 정확한 JSON 스키마 규격으로 반환해야 한다. | Must | AI |
| **REQ-FE-004** | STAR 하라이트 UI | 백엔드에서 받은 JSON 데이터를 바탕으로, 유저 답변 중 S, T, A, R에 해당하는 문장을 서로 다른 색상으로 하이라이팅하여 시각화한다. | Must | FE |
| **REQ-FE-005** | 원포인트 행동 개선 가이드 | 종합 점수 나열을 지양하고, 가장 누적 점수가 낮은 성분(예: Action 부재)을 콕 집어 "이 문장을 이렇게 고쳐보세요"라는 원포인트 코멘트를 노출한다. | Must | FE/AI |

## 3. 데이터베이스(DB) 필수 엔티티 구조

요구사항을 충족하기 위해 백엔드에서 설계해야 할 핵심 테이블 구조입니다.

### 3.1 `User` (사용자)
* `id` (PK)
* `email`
* `target_job` (목표 직무)
* `created_at`

### 3.2 `InterviewSession` (면접 세션)
* `id` (PK)
* `user_id` (FK)
* `company_name` (입력 기업명)
* `job_description` (입력 JD 텍스트)
* `created_at` *(하루 1회 생성 제한 검증용)*

### 3.3 `TurnHistory` (대화 턴 히스토리)
* `id` (PK)
* `session_id` (FK)
* `turn_index` *(0: 기본 질문, 1: 꼬리 1, 2: 꼬리 2)*
* `question` (AI 생성 질문)
* `question_meta` (출제 의도 근거 문장)
* `user_answer` (유저 입력 텍스트)
* `answer_summary` (토큰 절약용 답변 요약본)

### 3.4 `FeedbackReport` (최종 리포트)
* `id` (PK)
* `session_id` (FK)
* `star_json_data` *(S/T/A/R 컴포넌트별 매칭 텍스트 및 개별 점수)*
* `one_point_guide` (원포인트 행동 개선 문장)
* `is_retry` (재도전 세션 여부 블리언)
* `created_at`
